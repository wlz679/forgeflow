# i18n 中英文双语支持 — 设计文档

## 1. 背景与动机

- 当前项目仅英文，开发者自身英文不好，需要中文界面来维护和管理网站
- 中文面向开发者自身使用，**不需要 SEO**（Google 只需收录英文页面）
- 纯客户端切换，无 URL 路径变化，通过查询参数 `?lang=zh` 切换

## 2. 范围

### 包含（核心 UI + 数据）

- 所有 `.astro` 组件和页面中的静态 UI 文本（导航、按钮、标题、标签等）
- 30 个工具引擎的 title、description、input labels、FAQ（q/a）、howToUse steps
- 6 个分类的 name、description
- 首页、博客列表页的标题和描述
- 英文保持不变作为默认语言

### 不包含

- 30 篇博客文章正文
- 隐私政策、条款页面正文
- 引擎的 staticExamples（示例标题，SEO 内容，只需英文）
- 引擎 generate 函数的输出内容（运行时生成的结果，只需英文）
- sitemap、robots.txt（无需变更）
- `lang` 属性作为 URL 参数不影响 Astro 静态路由

## 3. 技术方案

### 3.1 语言切换机制

```
用户访问任意页面
  → 检查 URL 查询参数 ?lang=zh（优先级最高）
  → 若无，检查 cookie yt-lang
  → 若无，默认英文
  → 点击 Header 切换链接 → 设置 cookie + 重定向当前页 + ?lang=xx
```

- **URL 参数**：`?lang=zh`（查询参数，不影响路由）
- **Cookie**：`yt-lang=en|zh`（30 天过期，跨页面持久化）
- **默认**：英文（无参数 / 无 cookie 时）
- **不改变 Astro 路由**：静态生成路径不变

### 3.2 文件结构

```
src/i18n/
├── index.ts            # t(key, lang) 查询函数, getLang(Astro) 语言检测
└── translations.ts     # 翻译字典，扁平 key → { en, zh }
```

### 3.4 翻译函数与语言检测

```typescript
// src/i18n/index.ts
import { translations } from './translations';

export type Lang = 'en' | 'zh';

/** 从 Astro 上下文检测当前语言 */
export function getLang(astro: { url: URL; cookies: { get(name: string): { value: string } | undefined } }): Lang {
  const q = astro.url.searchParams.get('lang');
  if (q === 'zh' || q === 'en') return q;
  const c = astro.cookies.get('yt-lang');
  if (c && (c.value === 'zh' || c.value === 'en')) return c.value;
  return 'en';
}

/** 获取翻译文本，支持 {变量} 插值 */
export function t(key: string, lang: Lang, vars?: Record<string, string>): string {
  const entry = translations[key];
  if (!entry) return key; // fallback: 显示 key 本身
  let text = entry[lang];
  if (vars) {
    for (const [k, v] of Object.entries(vars)) {
      text = text.replace(`{${k}}`, v);
    }
  }
  return text;
}
```

### 3.5 Header 语言切换

```astro
<!-- 在 Header.astro 中新增语言切换链接 -->
<a href="?lang=zh" class="text-xs text-gray-400 hover:text-red-600">中文</a>
<!-- 当前已是中文时显示 English 链接 -->
```

切换逻辑：
1. 点击 "中文" 链接 → 当前页面 URL 加 `?lang=zh`
2. 页面加载时 `getLang()` 检测到 `?lang=zh` → 设置 cookie `yt-lang=zh`
3. 后续页面从 cookie 读取，无需额外参数
4. 点击 "English" → `?lang=en`

Cookie 设置在客户端完成（`<script>` 标签），避免引入 Astro middleware。

### 3.6 页面中使用方式

每个 `.astro` 页面的 frontmatter 中：

```astro
---
import { t, getLang } from '../i18n';
const lang = getLang(Astro);
---
<h1>{t('home.title', lang)}</h1>
<p>{t('home.subtitle', lang)}</p>
```

对于工具页 `[slug].astro`，引擎数据中的 title、description 也通过 `t()` 获取：

```astro
<h1>{t(`tools.${slug}.title`, lang)}</h1>
<p>{t(`tools.${slug}.description`, lang)}</p>
<label>{t(`tools.${slug}.input.${input.name}.label`, lang)}</label>
```

FAQ 和 HowToUse 组件接收的数据也通过翻译字典获取。

## 5. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 语言切换方式 | 查询参数 + Cookie | 中文不需要 SEO，简单即可 |
| URL 结构 | 不变（无路径前缀） | 避免 SEO 碎片化 |
| 翻译存储 | 单文件扁平 key 字典 | 项目规模小，集中管理方便 |
| 引擎文件 | 不改动 | 引擎的英文数据保持原样，翻译在组件层通过 t() 查字典完成 |
| Cookie 设置 | 客户端 JS | 避免引入 Astro middleware 增加复杂度 |
| 博客/法律页面 | 不翻译 | 开发者不看，只翻译核心 UI + 工具数据 |

## 6. 翻译估算

- UI 框架文本（导航、按钮、标签等）：约 30 条
- 6 个分类：12 条
- 30 个工具（title + description + input labels + howToUse steps + FAQ）：约 200-250 条
- 独立页面（首页、关于、联系、博客列表）：约 20 条
- **总计约 260-300 条翻译条目**
